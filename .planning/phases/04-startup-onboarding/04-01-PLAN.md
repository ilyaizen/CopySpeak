## Key Codebase Patterns

### Existing IPC Pattern (from commands/config.rs)
```rust
#[tauri::command]
pub fn get_config(config: State<'_, Mutex<AppConfig>>) -> AppConfig {
    config.lock().unwrap().clone()

```

### Config Path Detection (from config/mod.rs)
```rust
pub fn config_path() -> PathBuf {
    let base = dirs::config_dir().unwrap_or_else(|| PathBuf::from("."));
    base.join("CopySpeak").join("config.json")
}
```

  <name>task 3: Add first-run redirect in root layout</name>

  <action>
Modify the root layout to detect first-run and redirect to onboarding:

1. In `src/routes/+layout.svelte`, add to the existing onMount:
   - Check `invoke<boolean>("config_exists")` 
   - If false (no config), call `goto("/onboarding")` and return early
   - Do this BEFORE other initialization (appearance sync, event listeners)

2. Implementation:
```typescript
onMount(async () => {
  // First-run check - must be first
  try {
    const hasConfig = await invoke<boolean>("config_exists");
    if (!hasConfig) {
      goto("/onboarding");
      return;
    }
  } catch (e) {
    console.error("Failed to check config:", e);
  }
  
  // Existing initialization continues...
  await startHistoryEventListeners();
  // ... rest of existing code
});
```

3. Add import: `import { goto } from "$app/navigation";`

4. Note: The onboarding page itself does NOT need this check since user gets there via redirect, and skipping creates config which prevents re-redirect.

WHY: Checking in onMount (not +layout.ts load) avoids IPC race condition mentioned in roadmap. The redirect happens after Tauri is ready, not during SSR/load.
