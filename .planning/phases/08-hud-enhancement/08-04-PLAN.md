  <action>
Add a private `_emit` field to the store class, capture the `emit` function in `setupListeners()`, then call `this._emit?.("hud:stop", null)` in both `onended` and `handleStop()`.

**Step 1 — Add private field near other private fields at the top of the class:**
```typescript
private _emit: ((name: string, payload: unknown) => Promise<void>) | null = null;
```

**Step 2 — Update setupListeners() to capture emit:**
Change the destructuring from:
```typescript
const { listen } = await import("@tauri-apps/api/event");
```
To:
```typescript
const { listen, emit } = await import("@tauri-apps/api/event");
this._emit = emit;
```
This must be placed before any `await listen(...)` calls — `emit` is used at store level, not inside any listener.

**Step 3 — Update setAudioElement() onended handler:**
```typescript
el.onended = () => {
  this.isPlaying = false;
  this.isPaused = false;
  void this._emit?.("hud:stop", null);
};
```

**Step 4 — Update handleStop():**
```typescript
handleStop() {
  if (this._audioEl) {
    this._audioEl.pause();
    this._audioEl.currentTime = 0;
  }
  this.isPlaying = false;
  this.isPaused = false;
  void this._emit?.("hud:stop", null);
}
```

**Note on void:** `this._emit` returns a Promise. Using `void` prevents unhandled promise rejection warnings without needing await (both methods are synchronous). This is the correct pattern for fire-and-forget Tauri emits.

**Note on safety:** `_emit` is null before setupListeners() completes and in browser (non-Tauri) mode. The `?.` operator handles both cases gracefully — no HUD was ever shown in those cases anyway.

Do NOT add invoke("hide_hud") — the CSS opacity transition triggered by the event is sufficient per CONTEXT.md. If the OS-level window hide is needed, that is a separate concern.

Do NOT modify any other methods, fields, or imports beyond what is described above.
