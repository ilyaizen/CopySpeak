### Infrastructure Issue Details

**Root Cause:** The current vitest configuration (`vitest.config.ts`) is not properly configured for Svelte 5 component testing with `@testing-library/svelte@5.3.1`. The error `mount(...) is not available on server` indicates that components are being rendered in server mode instead of client mode.

**Evidence:**
- Existing test `src/lib/components/layout/app-header.test.ts` fails with the same error
- Non-component tests (`src/lib/models/html-templates.test.ts`) pass successfully
- Error occurs at the `render()` call from `@testing-library/svelte`

**Test Files Created (but not verified):**
1. `src/lib/components/engine/engine-tabs.test.ts` - 4 tests
2. `src/lib/components/engine/local-engine.test.ts` - 19 tests
3. `src/lib/components/engine/http-engine.test.ts` - 14 tests
4. `src/lib/components/engine/openai-engine.test.ts` - 14 tests
5. `src/lib/components/engine/elevenlabs-engine.test.ts` - 14 tests

**Total:** 65 tests generated, blocked by infrastructure issue.

### Recommendations

**To achieve Nyquist compliance:**
1. Fix vitest configuration for Svelte 5 component testing
2. Consider using experimental Svelte 5 mode in `@testing-library/svelte` if available
3. Verify that `compilerOptions: { css: 'injected' }` is compatible with Svelte 5
4. Check if `hot: !process.env.VITEST` option is interfering with test mode
5. Run `bun run test` after infrastructure fix to verify all 65 tests pass
