## Integration Gaps (Critical Blockers)

### Gap 1: CLI Preset Selection Doesn't Apply Command/Args (ENG-02)

**Files:** `src/lib/components/engine/local-engine.svelte` (line 116), `src/lib/components/engine/engine-tabs.svelte` (lines 123-131)

**Problem:** The preset dropdown's `onchange` handler only sets `localConfig.tts.preset` but does not update `localConfig.tts.command` or `localConfig.tts.args_template`. The `applyCliPreset()` function and `CLI_PRESETS` map exist in `engine-tabs.svelte` but are never called — `local-engine.svelte` has no mechanism to invoke parent functions.

**Impact:** User selects "Kokoro TTS" preset → only `localConfig.tts.preset = "kokoro-tts"` is set → command/args remain stale from previous preset → saving persists wrong configuration. The comment at line 117 explicitly acknowledges this: `// Apply preset configuration would be handled by parent`.

**Requirements affected:** ENG-02

---

### Gap 2: Nav Tabs Visible During Onboarding — Escape Path Without Saving Config (OBD-02, OBD-03)

**Files:** `src/routes/+layout.svelte` (line 89), `src/routes/onboarding/+page.svelte`

**Problem:** `+layout.svelte` unconditionally renders `<AppHeader />` (which includes all three nav tabs) around every route, including `/onboarding`. There is no `/src/routes/onboarding/+layout.svelte` to suppress or override the header. The onboarding page renders its full-screen card _below_ the visible nav tabs.

**Impact:**
1. User can click Play/Engine/Settings during onboarding → navigates away without calling `set_config`
2. No config file written to disk
3. Next app launch: `config_exists()` returns `false` → `goto("/onboarding")` → infinite redirect loop

**Requirements affected:** OBD-02, OBD-03
